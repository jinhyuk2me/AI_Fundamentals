## TL;DR
- 푸리에 변환은 신호를 주파수 도메인으로 옮겨 합성곱을 곱셈으로 단순화하며, Spectral CNN과 오디오 모델에 필수다.
- Phase 4의 첫 번째 목표로, 분석/신호 처리 노트의 허브가 되어 [[Math/Math_확장_계획]]에서 정의한 실습·링크 요구사항을 수용한다.
- 이 템플릿은 FFT 실습 코드, 스펙트럼 시각화, 컨볼루션 정리 증명 스케치를 넣을 자리를 확보한다.

## 언제 쓰나
- 음성, 음악, 시계열 데이터를 주파수 도메인 특성으로 분석하거나 증강할 때
- CNN을 FFT 기반 합성곱으로 가속하거나, Spectral Graph Convolution을 설계할 때
- Diffusion, Stable Audio 등 주파수 영역 손실을 쓰는 모델을 구현할 때

## 주요 API
| 이름 | 설명 | 예정 실습 |
| --- | --- | --- |
| `numpy.fft.fft` | 1D 이산 푸리에 변환 | 오디오 샘플 스펙트럼 분석 |
| `torch.fft.fftn` | N차원 FFT, GPU 가속 가능 | 2D 주파수 도메인 합성곱 구현 |
| `scipy.signal.stft` | 시간-주파수 해석(Short-Time Fourier Transform) | Spectrogram 시각화 |
| `torch.fft.irfft` | 실수 신호 역 FFT | 주파수 도메인 필터링 후 시간 영역 복원 |

## 실습 예제
1. UrbanSound 예제 클립을 로드해 `numpy.fft.fft`와 `torch.fft.fft` 결과를 비교하고, magnitude/phase를 별도 그래프로 표시 (작성 예정).
2. 2D 이미지에 FFT 기반 합성곱을 적용해 공간 도메인 대비 속도 차이를 표로 정리 (작성 예정).
3. Spectral Graph Convolution toy 예제를 구현해 라플라시안 고유값 기반 필터링 효과를 시각화 (작성 예정).

## 실수 주의
- 샘플링 간격과 FFT 길이를 혼동하면 주파수 해상도가 왜곡되므로 $\Delta t$와 $N$을 명확히 기록한다.
- `torch.fft`는 기본이 복소수 Tensor이므로 `.abs()`나 `.angle()` 호출 시 타입 변환을 확인해야 한다.
- STFT를 사용할 때 윈도 길이와 hop size를 부주의하게 설정하면 aliasing이 발생해 학습 데이터 품질이 저하된다.
